# AGENTS.md — 给 Codex / AI 协作者的操作手册

这是 San 的个人创意主页（GitHub Pages 静态站）。读完这个文件再动手。

## 站点结构（一共只有 4 个核心文件）

| 文件 | 作用 | 日常是否需要改 |
| --- | --- | --- |
| `data.json` | **唯一的内容数据源**：个人信息、创作方向、全部作品 | ✅ 几乎所有更新都只改它 |
| `index.html` | 唯一的页面骨架（单页） | 仅在新增版块时改 |
| `styles.css` | 唯一的样式文件（黑金编辑风） | 仅在调整视觉时改 |
| `app.js` | 唯一的脚本：读 `data.json` 渲染页面 | 仅在新增版块时改 |

没有别的页面，没有后台，没有数据库，没有构建步骤。保持这个状态。

## 最常见的任务：更新内容

**新增一个作品 / 记录**：在 `data.json` 的 `works` 数组开头加一条：

```json
{
  "date": "2026-06-15",
  "title": "作品标题",
  "type": "Codex 小工具",
  "description": "一两句话：解决了什么问题，做出了什么。",
  "status": "已完成",
  "featured": false
}
```

- `featured: true` 的作品会出现在首页「精选作品」区，全站保持 3–5 个精选即可。
- `status` 只用：`已完成` / `正在推进` / `持续更新`。
- 归档列表按日期自动倒序，不用手动排序。

**改个人介绍 / 联系方式**：改 `data.json` 的 `profile`。

**改创作方向**：改 `data.json` 的 `directions`（保持 4 个左右）。

## 风格红线（不要越界）

- 风格关键词：高级、克制、专业、编辑感、展览感。黑底金字（`--gold: #d8b36a`）。
- 不要游戏化：不用「升级 / 打怪 / 经验值 / 战力」这类词，不加进度条、XP、任务系统。
- 不要引入框架、构建工具、外部字体/CDN、后端、登录、数据库。
- 文案克制专业，不喊口号，不过度自我包装。

## 修改流程

1. 先判断：这次改动能不能只改 `data.json`？能就只改它。
2. 改页面/样式时做最小改动，保持现有组件语言一致。
3. 自测：本地 `python3 -m http.server` 打开 `index.html`（直接双击打不开，因为 fetch 需要 http），确认：
   - 页面正常渲染，控制台无报错；
   - `data.json` 是合法 JSON（`python3 -m json.tool data.json`）；
   - 手机宽度（≤640px）下排版正常。
4. 提交信息写清楚改了什么，推送到 `main` 后 GitHub Pages 自动发布。

## 完成后必须汇报

- 改了哪些文件、为什么
- 如何验证的
- 是否有遗留风险

如果用户的要求和本文件冲突，先指出冲突，再按用户当前指令执行。
